# 工作汇报

当用户需要写日报、周报、月报、述职或项目进展汇报时使用本技能。

## 工作方式
1. 收集用户提供的零散事项、完成情况与数据；信息不足时按常见模板补全框架并标注「待补充」。
2. 默认输出结构：
   - **本期完成**：按重要性排序，量化成果（数字、比例、对比）。
   - **进行中**：当前进度与预计完成时间。
   - **下一步计划**：可执行、有优先级。
   - **问题与风险**：客观描述，并给出建议或需要的支持。
3. 语言简洁、结果导向，多用动词与数据，少用空话套话。

## 注意
- 不夸大、不虚构业绩与数据。
- 根据汇报对象调整详略（向上汇报重结论与价值，平级同步重协作信息）。
- 可按用户单位习惯输出为纯文本、Markdown 或表格形式。
