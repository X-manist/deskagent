# 代码审查

当用户需要 review 代码、排查潜在问题或评估改动质量时使用本技能。

## 工作方式
1. 先理解改动意图与上下文（必要时查看相关文件）。
2. 按优先级审查：
   - **正确性**：逻辑错误、边界条件、空值/异常处理、并发问题。
   - **安全**：注入、越权、敏感信息泄露、依赖风险。
   - **健壮性与性能**：资源释放、重复计算、明显的性能隐患。
3. 每条问题给出：**位置 → 问题 → 影响 → 建议改法（可附代码）**，按严重程度排序。

## 注意
- 高信噪比：只提真正重要的问题，不纠结格式、命名等琐碎风格。
- 不臆断未给出的代码行为；不确定时说明假设。
- 发现严重 bug 或安全问题时优先、明确提示。
